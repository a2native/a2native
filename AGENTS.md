## Core purpose

a2native is a **human-in-the-loop (HITL) tool** for AI agents.

**Problem it solves:** Agents that need structured input from a human traditionally
use multi-turn chat loops (ask ‚Üí answer ‚Üí ask ‚Üí answer‚Ä¶). This is slow and error-prone.

**Solution:** The agent generates a JSON form spec and pipes it to `a2n`. A native
OS window (egui) appears; the user fills in fields and clicks Submit; the agent
receives a JSON result. One round-trip replaces many chat turns.

## Architecture

```
Agent (any language)
    ‚îÇ
    ‚îÇ  echo '{"components":[...]}' | a2n      ‚Üê one-shot
    ‚îÇ  a2n --session UUID                      ‚Üê multi-turn
    ‚îÇ  POST http://127.0.0.1:PORT/form         ‚Üê SSE daemon
    ‚îÇ  ws://127.0.0.1:PORT                     ‚Üê WebSocket daemon
    ‚ñº
  a2n binary (Rust, egui)
    ‚îú‚îÄ‚îÄ src/main.rs              CLI entry point (clap)
    ‚îú‚îÄ‚îÄ src/protocol/mod.rs      Protocol types (A2NInput, Component enum, etc.)
    ‚îú‚îÄ‚îÄ src/protocol/a2ui.rs     Google A2UI format parser
    ‚îú‚îÄ‚îÄ src/protocol/agui.rs     AG-UI envelope parser
    ‚îú‚îÄ‚îÄ src/renderer/mod.rs      Renderer trait
    ‚îú‚îÄ‚îÄ src/renderer/egui_impl/mod.rs        App + session daemon
    ‚îú‚îÄ‚îÄ src/renderer/egui_impl/components.rs Component rendering
    ‚îú‚îÄ‚îÄ src/session.rs           Port file management (TMPDIR)
    ‚îî‚îÄ‚îÄ src/server.rs            SSE + WebSocket daemon threads
```

### Three input formats (auto-detected, in priority order)

| Priority | Format | Detection | Output |
|---|---|---|---|
| 1 | **AG-UI** | contains `"TOOL_CALL_START"` | `TOOL_CALL_RESULT` event |
| 2 | **Google A2UI** | contains `surfaceUpdate` / `beginRendering` | `userAction` event |
| 3 | **a2native** (own format) | any other JSON | `{"status","values"}` |

### Session / daemon architecture

- `a2n --session UUID` ‚Äî first call spawns a background daemon (owns the egui window).
  Subsequent calls are short-lived clients connecting via TCP to the daemon.
- Port file: `{TMPDIR}/a2n-session-{UUID}.port`
- `a2n --close UUID` ‚Äî sends Close IPC message, daemon exits cleanly.
- SSE (`--sse PORT`) and WebSocket (`--ws PORT`) are optional listeners started by the daemon.
- This is the same client-daemon pattern used by agent-browser.

## Protocol schema

The authoritative schema is at:
- `schema/a2native-v0.1.schema.json` (in this repo)
- `https://a2native.github.io/schema/a2native-v0.1.schema.json` (hosted)
- `a2n schema` prints it locally

When adding new component fields, update **both** the Rust protocol types in
`src/protocol/mod.rs` AND the JSON schema in `schema/a2native-v0.1.schema.json`.

## Component rules

### Security banner ‚Äî NEVER remove or suppress

Every rendered window MUST display the amber security banner at the top:

> ‚öÝ This interface was generated by an AI agent ‚Äî do not enter sensitive
> information unless you trust the source.

- The banner is rendered in `update()` in `egui_impl/mod.rs`
- It CANNOT be suppressed via the form spec ‚Äî there is intentionally no field for it
- It is locale-aware (English, Chinese, Japanese, Korean)
- Do not add any mechanism that allows disabling or hiding the banner

### Validation (v0.1+)

Text input components (`text-field`, `textarea`, `password`) support:
- `required: bool` ‚Äî field must be non-empty on submit
- `min_length: int` ‚Äî minimum character count
- `max_length: int` ‚Äî maximum character count
- `pattern: string` ‚Äî ECMAScript-compatible regex the value must match
- `error_message: string` ‚Äî custom message shown on validation failure

Validation runs **on submit** (not live). Errors are displayed inline below the
field in red (`Color32::from_rgb(220, 60, 60)`). The form does NOT submit if any
validation fails.

### Image rendering

`image` components load from local file paths. URL loading is not supported.
If loading fails, fall back to displaying `[image: alt]`.

### Layout

- `row` ‚Äî renders children as equal-width horizontal columns via `ui.columns(n, ...)`
- `card` ‚Äî renders children in a bordered `Frame::group` with optional title

## Style rules

- Button primary color: `Color32::from_rgb(52, 120, 246)` (matches BTN_PRIMARY_BG constant)
- Accent color from `theme.accent_color` overrides selection, hyperlink, widget hover/active
- Dark/light mode from `theme.dark_mode`; defaults to system preference
- `apply_theme()` is called on EVERY frame (not just once) in `update()`
- Window default size: 600√ó500, resizable

## Binary / release

- The binary is cross-compiled for macOS ARM64, macOS x64, Linux x64, Linux ARM64, Windows x64
- Release workflow: push a `vX.Y.Z` tag ‚Üí GitHub Actions builds binaries, generates
  `SHA256SUMS.txt`, creates GitHub Release, publishes npm binary package
- crates.io: published as `a2native`; `cargo install a2native` installs the `a2n` binary

## What NOT to do

- Do NOT change the binary name from `a2n`
- Do NOT add any flag to suppress or bypass the security banner
- Do NOT add network access to the renderer (no phone-home, no telemetry)
- Do NOT remove the `branches-ignore: ['**']` guard in `release.yml` (prevents email spam)
- Do NOT commit secrets ‚Äî the `ORG_TOKEN` secret is set at org level in GitHub
- Do NOT use `NPM_TOKEN` ‚Äî the org-level secret is named `ORG_TOKEN`
- Do NOT add heavy GUI framework dependencies ‚Äî egui + rfd + clap is the intentional stack
- Do NOT use `regex` as a heavy dependency if a lighter alternative suffices ‚Äî but the
  `regex` crate is acceptable for pattern validation
